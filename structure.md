Project Structure GuideThis document provides a brief explanation of every folder and file in the Food Supply Chain Traceability System repository.Root DirectoryREADME.md: The main landing page for the repository. It contains the project overview, tech stack, setup instructions, and execution steps.docs/ (Documentation)Contains all the written academic deliverables and system planning documents.01_Problem_Statement.md: Defines the real-world issue the database solves.02_Objectives.md: Lists the academic and functional goals.03_Scope.md: Defines what is included and excluded from the project.04_Functional_Requirements.md: Details what the system must do (e.g., register batches, trigger alerts).05_Non_Functional_Requirements.md: Details system performance, security, and scalability goals.06_Business_Rules.md: The logical rules the database enforces (e.g., expiry > production date).07_ER_Diagram.md: Explains the Entity-Relationship model.08_Relational_Schema.md: Shows how the ER model translates into tables and keys.09_Normalization.md: Explains the step-by-step process of organizing tables into 3NF.10_System_Workflow.md: Describes the step-by-step lifecycle of a food batch.11_Use_Case.md: Describes what different users (actors) can do in the system.12_DFD.md: Data Flow Diagrams showing how information moves through the system.13_Data_Dictionary.md: A comprehensive list of all tables, columns, and data types.14_Project_Report.md: The final, compiled academic report mapping the project to the course rubric.diagrams/ (Visual Models)Contains the raw source code for rendering charts.er_diagram.mmd, use_case.mmd, workflow.mmd, dfd_level0.mmd, dfd_level1.mmd, relational_schema.mmd: Mermaid JS source files for generating the project diagrams.images/: A folder to store the exported PNG/JPG versions of the diagrams.database/ (SQL Implementation)Contains all modular PostgreSQL scripts required to build and test the system. They are numbered to be executed in exact order.01_create_database.sql: Initializes the empty PostgreSQL database.02_create_tables.sql: Creates the 3NF table structures.03_constraints.sql: Applies Primary Keys, Foreign Keys, and Check constraints.04_indexes.sql: Creates performance-enhancing indexes for queries.05_views.sql: Creates virtual tables (views) for easier reporting.06_insert_data.sql: Populates the tables with the mandatory 10 rows of sample data.07_single_table_queries.sql: Contains 10 standard DML queries on individual tables.08_multi_table_queries.sql: Contains 5 complex queries using JOINs.09_subqueries.sql: Contains various subquery demonstrations.10_functions.sql: Defines custom PL/pgSQL functions.11_procedures.sql: Defines PL/pgSQL stored procedures for transactions.12_triggers.sql: Contains event-driven PL/pgSQL logic (e.g., auto-quarantining).13_test_queries.sql: A script specifically to demonstrate that functions and triggers work.14_drop_all.sql: A cleanup script to safely delete all tables and objects.sample_output/An empty placeholder directory where you will save screenshots of your SQL queries running successfully in pgAdmin for your final submission.presentation/ (Defense Materials)Presentation_Outline.md: A slide-by-slide guide for your final project defense.Demo_Script.md: A script to follow while demonstrating the database live to your professor.appendix/ (Supplementary Info)References.md: Academic citations and sources used for the project.Glossary.md: Definitions of technical terms used in the documentation.Assumptions.md: Logical assumptions made during system design.
